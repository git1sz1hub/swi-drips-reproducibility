# SWI DRIPS Reproducibility Package

This repository reproduces the results and paper figures for the seawater intrusion DRIPS study. Three training batches are used (`stage_1`, `stage_2`, `stage_3`), with large-step DMD rank `r = 29` and time subsampling `sparseTime = 50`.

## Folder Structure

- `run_all.m` - main reproduction script.
- `scripts/` - MATLAB functions for PROM loading/rebuilding, DRIPS precomputation, plotting, and optional standard-DMD cache recomputation.
- `scripts/helpers/` - numerical helpers (`xDMD_affine`, IDW weights, matrix reshape utilities, QoI functions).
- `data/stage_1/` - first training batch.
- `data/stage_2/` - second training batch.
- `data/stage_3/` - third training batch.
- `data/heldout_cases/` - full 100-case held-out salinity test set and held-out parameter metadata.
- `data/results/` - cached error summaries and standard-DMD rank values used for figure reproduction.
- `roms/large_step_DMD/` - included large-step DMD PROMs.
- `figures/generated/` - output folder written by `run_all.m`.

Each stage directory contains:

- `parameters/parameter_samples.mat` - a `3 x N` variable named `parameters` with rows `[flux_rate; poro_mean; perm_mean]`.
- `salinity_splits/` - salinity snapshots used by MATLAB.

`data/heldout_cases/salinity_splits/` contains `salinity_01.mat` through `salinity_100.mat`. The held-out parameter list is in `data/heldout_cases/parameters/rand_samples.txt`.

## Requirements

- MATLAB R2024b or a recent MATLAB version with `exportgraphics`, `tiledlayout`, `maxk`, `mink`, `svd`, `schur`, and `funm`.
- No external MATLAB toolboxes are intentionally required.
- Disk footprint is about 22 GB.
- Default reproduction uses included PROM/error caches and took about 1 minute on the machine used to prepare this package. Optional cache recomputation can take substantially longer and requires enough memory for dense SVDs of `20000 x 1461` snapshot matrices.

## Large Data and Git LFS

The salinity split files are stored directly as `.mat` files under:

- `data/stage_1/salinity_splits/`
- `data/stage_2/salinity_splits/`
- `data/stage_3/salinity_splits/`
- `data/heldout_cases/salinity_splits/`

Many data files are larger than GitHub's normal 100 MiB Git limit or large enough to make regular Git pushes unreliable, so they are tracked with Git LFS. The included `.gitattributes` tracks the salinity split files and included PROM files with LFS.

After cloning, make sure Git LFS has downloaded the actual data files:

```bash
git lfs install
git lfs pull
```

## Run Reproduction

From MATLAB:

```matlab
cd('/path/to/swi_drips_reproducibility')
run_all
```

From a shell:

```bash
matlab -batch "cd('/path/to/swi_drips_reproducibility'); run_all"
```

The script writes regenerated figures to `figures/generated/`.

## Optional Recomputation

`run_all.m` defaults to the included PROMs and cached error summaries. To rebuild the large-step DMD PROMs before plotting, set this flag near the top of `run_all.m`:

```matlab
rebuildLargeStepPROMs = true;
```

The repository also includes:

- `rebuild_large_step_PROMs(cfg)` - rebuilds `roms/large_step_DMD/stage_*_PROMs.mat` and `training_errors_large_step_DMD.mat`.
- `determine_standard_DMD_ranks(cfg)` - recomputes `standard_DMD_rank_values.mat`.
- `recompute_standard_DMD_training_errors(cfg)` - recomputes standard-DMD full and coarse training-error caches.
